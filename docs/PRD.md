# GeneScope AI — Product Requirements Document (PRD)

> Status: Living document — last updated at Phase 12 (mobile frozen), entering Phase 13 (web app)

---

## 1. Project Summary

GeneScope AI is an **explainable, privacy-conscious, offline-resilient genetic risk
screening platform**. It predicts the likelihood of a genetic disorder category from
patient-reported clinical data, explains *why* the model reached that conclusion, and
converts that explanation into a plain-language summary.

This is a **portfolio/demo project** built on a public dataset — not real patient data,
and not a certified medical device.

---

## 2. Problem Statement

Most ML-powered health screening demos are black boxes: they output a number with no
reasoning, no privacy consideration, and no resilience when offline. GeneScope AI exists
to demonstrate that a screening tool can be:

- **Accurate** — a real trained model, not a hardcoded rule set
- **Explainable** — every prediction comes with the top contributing factors (SHAP)
- **Private by design** — no name, email, or phone ever leaves the device
- **Resilient** — works even without an internet connection
- **Human-centered** — always defers to real doctors, never claims to diagnose

## 3. Target Users

| User type | Why they'd use it |
|---|---|
| **End user (demo persona)** | Wants a quick, understandable genetic risk screening without creating an account or handing over personal data |
| **Recruiters / interviewers** | Evaluating real ML + full-stack + product thinking in a portfolio context |
| **Yuvak (owner)** | Portfolio centerpiece demonstrating ML, explainability, GenAI integration, privacy engineering, and product design skill |

---

## 4. Goals

- Predict genetic disorder risk category from structured clinical inputs
- Explain the top contributing factors behind each prediction (SHAP)
- Convert the explanation into a plain-language AI-generated summary
- Never collect or transmit personally identifying information
- Continue to function (in a reduced way) without internet access
- Persist anonymized prediction history tied to a hashed device identity
- Present all of the above in a polished, trustworthy, healthcare-grade UI

## 5. Non-Goals

- **This is not a diagnostic tool.** It never claims to diagnose, and always
  recommends consulting a real doctor.
- **This is not a medication or treatment advisor.** The AI Assistant is explicitly
  restricted from suggesting medications or treatment plans.
- **This is not a user-account system.** There is no login, no password, no stored
  real identity — persistence is achieved via an anonymous hashed device ID only.

## 7. Success Criteria

- A user can complete a full screening (form → prediction → explanation → summary)
  in under 2 minutes, with zero personal data ever transmitted
- The app clearly communicates it is not a diagnostic tool, without undermining trust
- The web app visually reads as a professional healthcare product, not a form demo
- The AI Assistant never gives medical advice beyond explaining the existing result
  and redirecting clinical questions to a doctor
- The project stands as a strong, honest, non-overclaiming portfolio centerpiece

---

## 8. Constraints

- No real patient data — public/synthetic dataset only
- No PII storage or transmission, anywhere, at any layer
- No medication names, dosages, or treatment plans generated by any AI component
- No fabricated features in documentation — every listed feature must reflect what is
  actually built, not aspirational claims
